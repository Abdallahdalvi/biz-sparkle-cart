# Mobile Layout & Button Placement Rule for Antigravity AI

## 🚨 CRITICAL RULE FOR ALL FUTURE TURNS AND CONVERSATIONS

The user has explicitly approved and mandated the following layout structure for all split sections on the website (e.g., Hero Section, Keypad Collection Section, and any future split text/image sections).

### 📱 Mobile View vs. Desktop View Mandatory Structure

1. **Desktop View (`lg:block` or `md:block`)**:
   - The call-to-action button (e.g., `VIEW CATALOG`, `SHOP THE KEYPADS`) MUST appear in the left text column, directly below the description/bullet points.

2. **Mobile View (`lg:hidden` or `md:hidden`)**:
   - The call-to-action button MUST NOT appear above the images.
   - The call-to-action button MUST appear **BELOW** the image section.

3. **Consistent Spacing**:
   - Ensure an equal, consistent gap (`pt-2` inside a `space-y-6` container or equivalent `pt-6` / `gap-6`) between the top text/images and the buttons across all sections.

### ❌ NEVER REVERT THIS LAYOUT
Do not move buttons back above images on mobile view under any circumstances unless explicitly requested by the user.
